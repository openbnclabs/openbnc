# openbnc

> Мост между криптоинфраструктурой и автономными AI-агентами.

---

## Что такое openbnc?

openbnc — это экспериментальный проект с открытым исходным кодом, исследующий, что происходит, когда автономные AI-агенты оказываются в центре криптоинфраструктуры.

Мы убеждены, что следующая глава децентрализованных технологий — это не просто перемещение средств между кошельками, а интеллектуальные агенты, способные рассуждать, действовать и сотрудничать от имени своих пользователей. openbnc — наша попытка построить этот мост: лёгкая, самохостируемая среда выполнения AI-агентов, созданная с нуля с прицелом на крипто-нативный мир.

Это не готовый продукт. Это эксперимент — и мы считаем, что самые интересные эксперименты проводятся публично.

---

## Видение

Криптоэкосистема создала выдающуюся инфраструктуру для безопасного обмена ценностями. AI-экосистема создала выдающуюся инфраструктуру для интеллектуальной автоматизации. Тем не менее эти два мира по-прежнему остаются во многом разделёнными, связанными лишь неудобными интеграциями и хрупкими обёртками.

openbnc задаёт простой вопрос: **что если AI-агенты станут гражданами первого класса в децентрализованной сети?**

Мы представляем будущее, в котором:

- Каждый может развернуть персонального AI-агента, действующего автономно от его имени
- Агенты могут нативно взаимодействовать с on-chain протоколами без централизованных посредников
- AI-вычисления и криптоэкономические стимулы согласуются, создавая самоподдерживающуюся экосистему
- Барьер между «запуском AI» и «участием в сети» исчезает полностью

Это амбициозное видение. Мы начинаем с малого, итерируем открыто и приглашаем сообщество помочь сформировать то, чем станет openbnc.

---

## Основные принципы

**Автономность по умолчанию.** Агенты, развёрнутые на openbnc, созданы для самостоятельных действий — не как пассивные чат-боты, ожидающие промптов, а как активные участники, способные планировать задачи, отслеживать условия и принимать осмысленные действия.

**Самохостинг и суверенитет.** Агент принадлежит вам. openbnc работает на вашем железе, вашем сервере или локальной машине. Никакой привязки к платформе. Никакого сбора данных третьими сторонами.

**Независимость от провайдера.** Мы не верим, что какая-то одна AI-модель будет лидировать вечно. openbnc поддерживает несколько AI-провайдеров, чтобы вы могли менять базовый интеллект, не перестраивая всю систему.

**Открытость и экспериментальность.** Мы делимся этой работой на раннем этапе, потому что считаем важными вопросы, которые ставит openbnc — и потому что лучшие ответы придут от сообщества, а не от одной команды.

### Сборка из исходников

```bash
git clone https://github.com/openbnclabs/openbnc.git
cd openbnc
cd web && npm install && npx vite build && cd ..
cargo build --release
```

### Быстрый старт (Ollama)

```bash
ollama pull qwen2.5:7b
PROVIDER=ollama API_KEY=http://localhost:11434 OPENBNC_MODEL=qwen2.5:7b ./openbnc daemon
```

Откройте **http://localhost:42617** в браузере.

### Интеграция с Telegram

```toml
[channels_config.telegram]
bot_token = "ВАШ_ТОКЕН_БОТА"
allowed_users = ["ваш_username"]
stream_mode = "partial"
interrupt_on_new_message = true
```

Используйте команду `daemon` (не `gateway`) для запуска.

### Разница между `gateway` и `daemon`

| Команда | Что запускается |
|---------|----------------|
| `openbnc gateway` | Только HTTP-шлюз |
| `openbnc daemon` | Шлюз + каналы + планировщик |
